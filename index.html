Dev Blog: Renderer Impl.

Intro

What is Rendering

Rendering is the process of generating images from a scene built up by 3D objects.

A common use case is in game engines like Unity and Unreal Engine:

Designers place 3D objects into a game scene, and the engine's rendering system processes this scene to deliver the final rendered images that players see in real-time.

Key Factors in a Scene for Rendering

What determines how a rendered image looks in a scene?

The main factors are the properties of the objects, the light sources, and the camera.

Object Properties

  • Texture: A texture is an image applied to the surface of an object. It‚Äôs like a skin that wraps around the object, made up of colored pixels.
  • Material: The material of an object defines how it interacts with light. It includes properties like how reflective, transparent, or shiny the object is. Textures are often part of the material, determining how much light is absorbed or reflected.
  • 3D Position: This tells us where the object is located in the scene. Knowing an object‚Äôs position relative to other objects, light sources, and the camera helps determine what is visible, how perspective works, and where shadows fall.

Light Sources

Light sources are important because they illuminate the scene, making objects visible in the rendered image. Without lights, everything would appear completely dark.

There are different types of light sources, each affecting the scene in unique ways. Here‚Äôs a simple example using a point light:

  • Position: This determines where the light is located in the scene. It‚Äôs like placing a light bulb in a room.
  • Color: This defines the color of the light emitted.

Camera Settings

In a 3D environment, the camera is positioned at a specific point in world coordinates.

An intuitive analogy is that the camera acts like human eyes, fixed at one location, looking out at a scene through a window frame (the image plane). The image that appears on this "window" is the rendered view of the scene as seen from the camera's position.

Two important factors determine how the scene is captured:

  • Camera Orientation: In most rendering systems, the camera is centered within the frame, so this is typically not a concern.

From these factors, we can tell the Field of View (FoV) and Aspect Ratio.

Coursework 1: Ray Tracer

Step 1: Ray Generation for Each Pixel

For each pixel on the image plane, create an inverse ray from the camera pointing to the pixel.

Ray getFragCoordRay(const vec2 frag_coord)
{
	// camera plane distance:
	float sensorDistance = 1.0; 
	
	// plane size min:
	vec2 sensorMin = vec2(-1, -0.5); 
	
	// plane size max:
	vec2 sensorMax = vec2(1, 0.5); 
	
	// the size of a single pixel wrt the image plane:
	vec2 pixelSize = (sensorMax - sensorMin) / vec2(viewport.x, viewport.y); 
	
	// pixel mapping:
	vec2 plane_coord = vec2(sensorMin + pixelSize * frag_coord)
	
	// camera position, ray origin:  (+:camera|0:plane|-:objects)
	vec3 origin = vec3(0, 0, sensorDistance);
	
	// inverse ray direction:
	vec3 direction = normalize(vec3(plane_coord, -sensorDistance));
	
	// inverse ray:
	return Ray(origin, direction);
}

Step 2: Tracing the Ray

We trace the inverse light ray back to the scene until it either hits an object or misses everything.

If a ray hits a point on the scene, we shade the corresponding pixel with the color on that hitting point.

There are three properties of the material of the hitting point that decides the color:

Step 5: Make More Realistic: Fresnel [fray-nel] Effect

The image with reflection and refraction looks okay, but the glass likely does not look like glass as it shows reflection and refraction at equal strength all over the sphere.
In reality, reflection and refraction strength vary with the view direction and the normal at the hit
point. The reflection is typically strong on grazing angles close to the edge of the sphere, while the refraction is strongest in the center.

To simulate this in computer graphics, we apply the Fresnel effect. The Fresnel effect ensures that the sum of the reflection and refraction weights is always ‚â§1\leq 1‚â§1Ôªø. Here, we simply assume:
